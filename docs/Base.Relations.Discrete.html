### <a id="discrete-relations">Discrete Relations</a>

This is the [Base.Relations.Discrete][] module of the [Agda Universal Algebra Library][].

We begin with a definition that is useful for defining poitwise "equality" of functions
with respect to a given "equality" relation (see also the definition of `_‚âàÃá_` in the [Base.Adjunction.Residuation][] module).

\begin{code}

module _ {A : Type Œ±} where

 PointWise : {B : Type Œ≤ } (_‚âã_ : BinRel B œÅ) ‚Üí BinRel (A ‚Üí B) _
 PointWise {B = B} _‚âã_ = Œª (f g : A ‚Üí B) ‚Üí ‚àÄ x ‚Üí f x ‚âã g x

\end{code}

Thus, given a binary relation `‚âã` on ‚ÄµB`, and a pair of functions `f, g : A ‚Üí B`,
we have `f (Pointwise _‚âã_) g` provided `‚àÄ x ‚Üí f x ‚âã g x`.

Here is the analogous definition for dependent functions.

\begin{code}

 depPointWise :  {B : A ‚Üí Type Œ≤ }
                 (_‚âã_ : {Œ≥ : Level}{C : Type Œ≥} ‚Üí BinRel C œÅ)
  ‚Üí              BinRel ((a : A) ‚Üí B a) _
 depPointWise {B = B} _‚âã_ = Œª (f g : (a : A) ‚Üí B a) ‚Üí ‚àÄ x ‚Üí f x ‚âã g x

\end{code}

Next we define a type that is useful for asserting that the image of a function
is contained in a particular "subset" (predicate) of the codomain.

\begin{code}

 Im_‚äÜ_ : {B : Type Œ≤} ‚Üí (A ‚Üí B) ‚Üí Pred B œÅ ‚Üí Type (Œ± ‚äî œÅ)
 Im f ‚äÜ S = ‚àÄ x ‚Üí f x ‚àà S

\end{code}


#### <a id="operation-symbols-unary-relations-binary-relations">Operation symbols, unary relations, binary relations</a>

The unary relation (or "predicate") type is imported from Relation.Unary of the [Agda Standard Library][].

```agda
Pred : ‚àÄ {a} ‚Üí Type a ‚Üí (‚Ñì : Level) ‚Üí Type (a ‚äî suc ‚Ñì)
Pred A ‚Ñì = A ‚Üí Type ‚Ñì
```
We represent "sets" as inhabitants of such predicate types.

(In the definition of `Pred` above, we replaced `Set` with `Type` for consistency with our notation.)

Sometimes it is useful to obtain the underlying type (`A`) over which the predicates in `Pred A ‚Ñì` (the "subsets" of `A`) are defined.

\begin{code}

 PredType : Pred A œÅ ‚Üí Type Œ±
 PredType _ = A

\end{code}

The binary relation types are called `Rel` and `REL` in the standard library, but we
will call them `BinRel` and `BinREL` and reserve the names `Rel` and `REL` for the relation
types we define below and in the [Base.Relations.Continuous][] module.

We import the "heterogeneous" binary relation type from the standard library and renamed `BinREL`.

```agda
BinREL : ‚àÄ {‚Ñì} (A B : Type ‚Ñì) (‚Ñì' : Level) ‚Üí Type (‚Ñì-max ‚Ñì (‚Ñì-suc ‚Ñì'))
BinREL A B ‚Ñì' = A ‚Üí B ‚Üí Type ‚Ñì'
```

A special case, the homogeneous binary relation type is also imported and renamed `BinRel`.

```agda
BinRel : ‚àÄ{‚Ñì} ‚Üí Type ‚Ñì ‚Üí (‚Ñì' : Level) ‚Üí Type (‚Ñì ‚äî lsuc ‚Ñì')
BinRel A ‚Ñì' = REL A A ‚Ñì'
```

Occasionally it is useful to extract the universe level over which a binary relation is defined.

\begin{code}

 Level-of-Rel : {‚Ñì : Level} ‚Üí BinRel A ‚Ñì ‚Üí Level
 Level-of-Rel {‚Ñì} _ = ‚Ñì
\end{code}


#### <a id="kernels">Kernels</a>

The *kernel* of `f : A ‚Üí B` is defined informally by `{(x , y) ‚àà A √ó A : f x = f y}`.
This can be represented in type theory and Agda in a number of ways, each of which
may be useful in a particular context. For example, we could define the kernel
to be an inhabitant of a (binary) relation type, or a (unary) predicate type.

\begin{code}

module _ {A : Type Œ±}{B : Type Œ≤} where

 ker : (A ‚Üí B) ‚Üí BinRel A Œ≤
 ker g x y = g x ‚â° g y

 kerRel : {œÅ : Level} ‚Üí BinRel B œÅ ‚Üí (A ‚Üí B) ‚Üí BinRel A œÅ
 kerRel _‚âà_ g x y = g x ‚âà g y

 kernelRel : {œÅ : Level} ‚Üí BinRel B œÅ ‚Üí (A ‚Üí B) ‚Üí Pred (A √ó A) œÅ
 kernelRel _‚âà_ g (x , y) = g x ‚âà g y

 open IsEquivalence

 kerRelOfEquiv :  {œÅ : Level}{R : BinRel B œÅ}
  ‚Üí               IsEquivalence R ‚Üí (h : A ‚Üí B) ‚Üí IsEquivalence (kerRel R h)

 kerRelOfEquiv eqR h = record  { refl = refl eqR
                               ; sym = sym eqR
                               ; trans = trans eqR
                               }

 kerlift : (A ‚Üí B) ‚Üí (œÅ : Level) ‚Üí BinRel A (Œ≤ ‚äî œÅ)
 kerlift g œÅ x y = Lift œÅ (g x ‚â° g y)

 ker' : (A ‚Üí B) ‚Üí (I : Type ùì•) ‚Üí BinRel (I ‚Üí A) (Œ≤ ‚äî ùì•)
 ker' g I x y = g ‚àò x ‚â° g ‚àò y

 kernel : (A ‚Üí B) ‚Üí Pred (A √ó A) Œ≤
 kernel g (x , y) = g x ‚â° g y

-- The *identity relation* (equivalently, the kernel of a 1-to-1 function)
0[_] : (A : Type Œ±) ‚Üí {œÅ : Level} ‚Üí BinRel A (Œ± ‚äî œÅ)
0[ A ] {œÅ} = Œª x y ‚Üí Lift œÅ (x ‚â° y)

module _ {A : Type (Œ± ‚äî œÅ)} where

 -- Subset containment relation for binary realtions
 _‚äë_ : BinRel A œÅ ‚Üí BinRel A œÅ ‚Üí Type (Œ± ‚äî œÅ)
 P ‚äë Q = ‚àÄ x y ‚Üí P x y ‚Üí Q x y

 ‚äë-refl : Reflexive _‚äë_
 ‚äë-refl = Œª _ _ z ‚Üí z

 ‚äë-trans : Transitive _‚äë_
 ‚äë-trans P‚äëQ Q‚äëR x y Pxy = Q‚äëR x y (P‚äëQ x y Pxy)
\end{code}


### <a id="compatibility-of-operations-and-relations">Compatibility of operations and relations</a>

Recall, from the [Overture.Signatures][] and [Overture.Operations][] modules which established
our convention of reserving the sybmols `ùìû` and `ùì•` for types that
represent operation symbols and arities, respectively.

In the present subsection, we define types that are useful for asserting and proving
facts about *compatibility* of operations and relations

\begin{code}

-- lift a binary relation to the corresponding `I`-ary relation.

eval-rel : {A : Type Œ±}{I : Type ùì•} ‚Üí BinRel A œÅ ‚Üí BinRel (I ‚Üí A) (ùì• ‚äî œÅ)
eval-rel R u v = ‚àÄ i ‚Üí R (u i) (v i)

eval-pred : {A : Type Œ±}{I : Type ùì•} ‚Üí Pred (A √ó A) œÅ ‚Üí BinRel (I ‚Üí A) (ùì• ‚äî œÅ)
eval-pred P u v = ‚àÄ i ‚Üí (u i , v i) ‚àà P

\end{code}

If `f : Op I` and `R : Rel A Œ≤`, then we say `f` and `R` are *compatible* just in case `‚àÄ u v : I ‚Üí A`, `Œ† i Íûâ I , R (u i) (v i)  ‚Üí  R (f u) (f v)`.

\begin{code}

_preserves_ : {A : Type Œ±}{I : Type ùì•} ‚Üí Op A I ‚Üí BinRel A œÅ ‚Üí Type (Œ± ‚äî ùì• ‚äî œÅ)
f preserves R  = ‚àÄ u v ‚Üí (eval-rel R) u v ‚Üí R (f u) (f v)

--shorthand notation for preserves
_|:_ : {A : Type Œ±}{I : Type ùì•} ‚Üí Op A I ‚Üí BinRel A œÅ ‚Üí Type (Œ± ‚äî ùì• ‚äî œÅ)
f |: R  = (eval-rel R) =[ f ]‚áí R

-- predicate version of the compatibility relation
_preserves-pred_ : {A : Type Œ±}{I : Type ùì•} ‚Üí Op A I ‚Üí Pred ( A √ó A ) œÅ ‚Üí Type (Œ± ‚äî ùì• ‚äî œÅ)
f preserves-pred P  = ‚àÄ u v ‚Üí (eval-pred P) u v ‚Üí (f u , f v) ‚àà P

_|:pred_ : {A : Type Œ±}{I : Type ùì•} ‚Üí Op A I ‚Üí Pred (A √ó A) œÅ ‚Üí Type (Œ± ‚äî ùì• ‚äî œÅ)
f |:pred P  = (eval-pred P) =[ f ]‚áí Œª x y ‚Üí (x , y) ‚àà P


-- The two types just defined are logically equivalent.
module _ {A : Type Œ±}{I : Type ùì•}{f : Op A I}{R : BinRel A œÅ} where
 compatibility-agreement : f preserves R ‚Üí f |: R
 compatibility-agreement c {x}{y} Rxy = c x y Rxy
 compatibility-agreement' : f |: R ‚Üí f preserves R
 compatibility-agreement' c = Œª u v x ‚Üí c x
\end{code}
